# notes/ — per-task working notes

One file per task: `notes/<yyyy-mm-dd>-<task-slug>.md`. Created at task start, updated after every PR. Purpose: the next session (or the reviewer) sees approach, dead ends, and open threads without re-deriving them.

Template:

```markdown
# <task title>
- **Tier**: S/M/L (per .claude/rules/effort.md)
- **State**: in progress / blocked / done
- **Approach**: <2-3 lines>
- **Dead ends**: <what was tried and abandoned, so nobody retries it>
- **Open threads**: <what's left / follow-ups>
- **PRs/commits**: <refs>
```

Housekeeping: when a task is done and reported, the note stays (cheap history); if it produced a durable lesson, that lesson goes to the knowledge files — not here (see `.claude/rules/memory.md`).
