## 3. עץ התהליך (איך עבודה זורמת — Iron Rule #13)

```
Board Sync (ג'ף מכנס)
  │
  ├─ כל ראש צוות מדווח מה הצוות שלו עשה בפועל (diff/commit, לא הצהרה)
  ├─ הבורד דן מה צריך לקרות קדימה
  ├─ ג'ף כותב סיכום לכרמל
  │
  ▼
שיגור עבודה במקביל (לא בתור) — ג'ף → כמה ראשי-צוות/סוכנים בבת אחת
  │
  ▼
כל סוכן עובד ב-worktree מבודד משלו
  │
  ▼
ג'ף בודק diff → מאמת בדפדפן אמיתי (Playwright, Iron Rule #9) → מוזג ל-main
  │
  ▼
activity_log.md מתעדכן (סטטוס סופי + הוכחה)
  │
  └──→ חוזר ל-Board Sync
```

**אישורים (לא דרך הדירקטוריון):** צוות → ראש צוות ישיר ← דירקטיבת בורד מאושרת. רק אסטרטגיה/בלתי-הפיך/כספים/קונפליקט לא-פתור עולים לכרמל/מעיין, ודרך ג'ף בלבד.
